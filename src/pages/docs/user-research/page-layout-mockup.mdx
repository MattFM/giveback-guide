# Page Layout Mock-up: Furahaa Project

**Last Updated:** October 21, 2025 (Updated with sidebar, ad placement, and support CTAs)  
**Purpose:** Visual representation of proposed layout changes using real project data  
**Related:** [Layout Design Survey](./layout-design-survey) | [Design Decisions](./project-stay-layout-decisions)

---

## Overview

This mock-up shows the proposed layout for Project pages using the real Furahaa project as an example. Furahaa is a plant-based fast food restaurant in Paris that employs deaf people, demonstrating our typical project structure.

**Project details:**
- **Name:** Furahaa
- **Location:** Paris, France
- **Type:** Cafe/Restaurant
- **Impact Areas:** Education & Skills Training (C02), Sustainable Food Systems (E03)
- **Status:** Not verified (typical case)

### Sidebar Behavior

**Sticky Positioning:**
- Sidebar starts at `position: sticky; top: 100px`
- Sticks when user scrolls past hero image
- Action bar and support CTA remain visible during scroll
- Unsticks before footer (doesn't overlap)

**Width Ratio:**
- Main content: 70% width
- Sidebar: 30% width
- Gap between: 2rem

**Tablet (768-1023px):**
- No sidebar
- All content flows vertically in single column
- Action bar appears at top (non-sticky)
- Support CTA appears in main flow

## Stay Page Differences

For Stay pages (e.g., a hotel), the layout is identical EXCEPT:

### Action Bar Changes

```
┌─────────────────┐  ┌──────────────────┐  ┌───────────────────┐
│  💾 Save to List│  │  🗺️ View on Map  │  │  🏨 Book Now ▾    │
└─────────────────┘  └──────────────────┘  └───────────────────┘
```

**Rationale:**
- "View on Map" is primary (location matters for accommodations)
- Direct website link doesn't earn commission (deprioritized)
- "Book Now" expands to show 6+ booking platforms

### Booking Options Expansion

```
EXPANDED "Book Now":
┌──────────────────────────────────────────────────────────────┐
```

## Comparison: Current vs. Proposed

### Current Layout Issues
- ❌ Impact Areas buried mid-page after About
- ❌ Get Involved box combines education + actions
- ❌ Metadata as plain text list
- ❌ Flat hierarchy, equal visual weight
- ❌ Inconsistent container usage
- ❌ Mobile action access requires scrolling

### Proposed Layout Improvements
- ✅ Impact Areas elevated to position 4 (after title)
- ✅ Actions separated: Bar at top + Education lower
- ✅ Metadata as interactive, scannable pills
- ✅ Three-tier visual system creates hierarchy
- ✅ Consistent container rules (actionable vs. informational)
- ✅ Quick action access on mobile (top bar or sticky)
