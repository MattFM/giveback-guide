# Color System & Hierarchy

Our color system follows the **60-30-10 rule** to create visual hierarchy and guide user attention to the most important actions on each page.

## The 60-30-10 Rule

- **60%** - Neutral base (backgrounds, body text)
- **30%** - Secondary/supporting color (sections, cards, borders)
- **10%** - Accent/action colors (CTAs, highlights, important UI elements)

## Color Palette

### Neutrals (90% of UI)

```css
:root {
  --color-background: #FFFFFF;     /* Main backgrounds */
  --color-surface: #F9FAFB;        /* Cards, sections */
  --color-text: #111827;           /* Primary text */
  --color-text-muted: #6B7280;     /* Secondary text */
}
```

### Primary Accent (10% - CTAs & Highlights)

```css
:root {
  --color-accent: #059669;         /* Emerald green - primary actions */
  --color-accent-hover: #047857;   /* Darker on hover */
  --color-accent-light: #D1FAE5;   /* Light tint for focus rings */
}
```

**Why Green?**
- Signals growth, nature, sustainability - aligned with our "giveback" mission
- Differentiates us from typical travel sites (which use blue)
- Creates positive emotional connection to environmental/social impact
- Excellent accessibility when dark enough

### Impact Category Colors (2% - Badges Only)

```css
:root {
  --color-impact-environmental: #059669;  /* Green */
  --color-impact-social: #DC6B4A;         /* Terracotta */
  --color-impact-cultural: #7C3AED;       /* Purple */
}
```

Use these **sparingly** - only for small badges/tags on project and stay cards.

## Button Hierarchy

### Primary Buttons

Use for the **most important action** in each context:
- Homepage: "Join our community"
- Project pages: "Visit Website"
- Stay pages: "Book Now" or "Visit Website"
- Header: "Join" / "Dashboard"

```astro
<Button href="/join" variant="primary">
  Join our community
</Button>
```

**Visual style:**
```css
.btn-primary {
  background: var(--color-accent);
  color: white;
  &:hover {
    background: var(--color-accent-hover);
  }
}
```

### Secondary Buttons

Use for supporting actions:
- "Save for Later"
- "Share"
- "Learn More"

```astro
<Button href="/save" variant="secondary">
  Save for Later
</Button>
```

**Visual style:**
```css
.btn-secondary {
  background: white;
  border: 2px solid var(--color-accent);
  color: var(--color-accent);
  &:hover {
    background: var(--color-accent);
    color: white;
  }
}
```

### Tertiary/Ghost Buttons

Use for low-priority actions:
- "Cancel"
- "Skip"
- Navigation links

```astro
<Button href="/cancel" variant="ghost">
  Cancel
</Button>
```

## Context-Based Hierarchy

### Global vs. Contextual CTAs

The site has **two layers of hierarchy**:

#### Global Navigation Layer (Header)
- Join/Dashboard button uses **primary accent color**
- Always visible, represents site-wide conversion goal
- Remains consistent across all pages

#### Page Content Layer
- Each page has its own primary CTA
- Uses the **same primary accent color**
- Differentiated by size and position, not color

**Why this works:**
- Different visual contexts (header chrome vs. content flow)
- Size indicates importance (header button smaller, page CTA larger)
- No confusion because they serve different purposes

### Example: Project Page

```astro
---
import Button from '../components/ui/Button.astro';
---

<!-- Header has global CTA (small) -->
<header>
  <Button href="/join" size="sm">Join</Button>
</header>

<!-- Page content has contextual CTA (large) -->
<main>
  <div class="hero">
    <h1>Amazing Wildlife Sanctuary</h1>
    <p>Help protect endangered species...</p>
    
    <div class="actions">
      <!-- Primary action for this page -->
      <Button href={project.websiteUrl} size="lg" variant="primary">
        Visit Project Website
      </Button>
      
      <!-- Secondary actions -->
      <Button variant="secondary">
        Save for Later
      </Button>
    </div>
  </div>
</main>
```

## Button Component

Create a reusable Button component at `src/components/ui/Button.astro`:

```astro
---
interface Props {
  href?: string;
  variant?: 'primary' | 'secondary' | 'ghost';
  size?: 'sm' | 'md' | 'lg';
  class?: string;
}

const { 
  href = '#', 
  variant = 'primary', 
  size = 'md', 
  class: extra = '' 
} = Astro.props;

const sizes = {
  sm: 'text-sm px-4 py-2',
  md: 'text-sm px-5 py-2.5',
  lg: 'text-base px-6 py-3.5'
};

const variants = {
  primary: 'text-white bg-accent hover:bg-accent-hover focus:ring-4 focus:ring-accent-light',
  secondary: 'bg-white border-2 border-accent text-accent hover:bg-accent hover:text-white focus:ring-4 focus:ring-accent-light',
  ghost: 'text-gray-600 hover:text-accent hover:underline'
};

const classes = `inline-flex items-center justify-center font-medium rounded-lg focus:outline-none transition-colors ${sizes[size]} ${variants[variant]} ${extra}`;
---

<a href={href} class={classes}>
  <slot />
  <slot name="icon" />
</a>
```

### Usage Examples

```astro
<!-- Primary CTA with icon -->
<Button href="/join" variant="primary" size="lg">

<!-- Secondary action -->

  Save for Later
```

## Impact Badges

Use category colors only for small badges on cards:

```astro
---
const impactColors = {
  'Environmental & Conservation': 'bg-impact-environmental',
  'Social Impact': 'bg-impact-social',
  'Cultural & Wellbeing': 'bg-impact-cultural'
};

const badgeColor = impactColors[category] || 'bg-gray-600';
---

<span class={`inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-medium text-white ${badgeColor}`}>
  {category}
</span>
```

**Important:** Never use impact colors for large surfaces or primary CTAs. They're visual accents only.

## Accessibility

### Color Contrast

All color combinations meet WCAG AA standards:

- **Primary Accent (#059669) on white**: 3.66:1 ✅
- **Accent text on accent bg**: White text ensures 7.5:1+ ✅
- **Text colors**: Gray-900 on white provides 16:1 ✅

### Don't Rely on Color Alone

Always pair color with:
- **Icons** for different action types
- **Text labels** that clearly indicate purpose
- **Position** to show hierarchy
- **Size** for emphasis

## Dark Mode

Adjust colors for dark mode:

```css
.dark {
  --color-background: #111827;
  --color-surface: #1F2937;
  --color-text: #F9FAFB;
  --color-text-muted: #9CA3AF;
  
  /* Accent stays the same or slightly lighter */
  --color-accent: #10B981;
  --color-accent-hover: #059669;
}
```

## Migration Guide

### Updating Existing Buttons

Replace inline button styles:

```diff
- <a href="/join" class="inline-flex items-center text-white bg-black hover:bg-primary-800 focus:ring-4 focus:ring-primary-300 font-medium rounded-lg text-sm px-5 py-2.5">
-   Join our community
- </a>

+ <Button href="/join" variant="primary">
+   Join our community
+ </Button>
```

### Finding Buttons to Update

Search for common button patterns:

```bash
# Find inline button classes
grep -r "bg-black.*hover:" src/pages/
grep -r "bg-primary" src/pages/

# Find CTA links
grep -r "href=\"/join\"" src/
grep -r "Visit Website" src/
```

## Summary

**How many colors?** 
- 1 primary accent (green) + neutrals
- Optional: 3 impact category colors for tiny badges only

**When to use accent color?**
- Primary CTA in each context
- Important highlights
- Interactive elements on hover/focus

**What makes hierarchy?**
- Context (global vs. page)
- Size (sm/md/lg)
- Position (header vs. content)
- Variant (primary/secondary/ghost)

**Key principle:** Hierarchy shifts by context. Your accent color follows the user's primary goal on each page, while maintaining global consistency.
